The Users endpoint group exposes the full set of ASP.NET Core Identity HTTP API endpoints via `MapIdentityApi<ApplicationUser>()`, plus a custom `/logout` endpoint. Together these endpoints let clients register new accounts, obtain JWT bearer tokens for use with all other API endpoints, refresh expired tokens, and sign out.

You do not need to implement any of this code yourself — it is generated by the ASP.NET Core framework and wired up in `Users.cs` with a single call.

---

## POST /api/Users/register

Creates a new user account. The `email` is used as both the username and the email address.

**Authentication:** not required  
**Returns:** `200 OK` on success

### Request body

<ParamField body="email" type="string" required>
  The email address for the new account. Used as the login identifier.
</ParamField>

<ParamField body="password" type="string" required>
  The account password. Must meet ASP.NET Core Identity's default complexity requirements: at least one uppercase letter, one lowercase letter, one digit, and one non-alphanumeric character.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://localhost:5001/api/Users/register \
      -H "Content-Type: application/json" \
      -d '{"email": "you@example.com", "password": "YourP@ssword1!"}'
    ```
  </Tab>
  <Tab title="Response">
    **200 OK** — empty body. The account is created and ready for login.
  </Tab>
  <Tab title="Error response">
    **400 Bad Request** — if the email is already taken or the password does not meet requirements:

    ```json
    {
      "type": "https://tools.ietf.org/html/rfc9110#section-15.5.1",
      "title": "One or more validation errors occurred.",
      "status": 400,
      "errors": {
        "DuplicateUserName": ["Username 'you@example.com' is already taken."]
      }
    }
    ```
  </Tab>
</Tabs>

## POST /api/Users/login

Authenticates a user and returns a JWT access token and a refresh token. Pass the `accessToken` in the `Authorization: Bearer` header for all subsequent authenticated requests.

**Authentication:** not required  
**Returns:** `200 OK` with token data

### Request body

<ParamField body="email" type="string" required>
  The registered email address.
</ParamField>

<ParamField body="password" type="string" required>
  The account password.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://localhost:5001/api/Users/login \
      -H "Content-Type: application/json" \
      -d '{"email": "you@example.com", "password": "YourP@ssword1!"}'
    ```
  </Tab>
  <Tab title="Response">
    **200 OK**

    ```json
    {
      "tokenType": "Bearer",
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "expiresIn": 3600,
      "refreshToken": "CfDJ8O..."
    }
    ```
  </Tab>
  <Tab title="Error response">
    **401 Unauthorized** — if the credentials are incorrect.
  </Tab>
</Tabs>

### Response fields

<ResponseField name="tokenType" type="string">
  Always `"Bearer"`.
</ResponseField>

<ResponseField name="accessToken" type="string">
  The JWT bearer token to include in the `Authorization` header of authenticated requests.
</ResponseField>

<ResponseField name="expiresIn" type="integer">
  Lifetime of the access token in seconds (default: 3600).
</ResponseField>

<ResponseField name="refreshToken" type="string">
  An opaque token used to obtain a new access token without requiring the user to log in again. Pass this to `POST /api/Users/refresh`.
</ResponseField>

## POST /api/Users/refresh

Exchanges a valid refresh token for a new access token and refresh token pair. Use this to silently renew a session when the access token has expired.

**Authentication:** not required  
**Returns:** `200 OK` with new token data (same shape as the login response)

### Request body

<ParamField body="refreshToken" type="string" required>
  The refresh token returned by a previous login or refresh call.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://localhost:5001/api/Users/refresh \
      -H "Content-Type: application/json" \
      -d '{"refreshToken": "CfDJ8O..."}'
    ```
  </Tab>
  <Tab title="Response">
    **200 OK**

    ```json
    {
      "tokenType": "Bearer",
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "expiresIn": 3600,
      "refreshToken": "CfDJ8P..."
    }
    ```
  </Tab>
</Tabs>

## POST /api/Users/logout

Signs out the currently authenticated user by clearing the authentication cookie. This endpoint is only relevant when using cookie-based authentication.

**Authentication:** required  
**Returns:** `200 OK` on success, `401 Unauthorized` if the request body is null

### Request body

Send any non-null JSON value as the body. An empty object `{}` is sufficient.

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://localhost:5001/api/Users/logout \
      -H "Authorization: Bearer <token>" \
      -H "Content-Type: application/json" \
      -d '{}'
    ```
  </Tab>
  <Tab title="Response">
    **200 OK** — empty body. The session cookie is cleared.
  </Tab>
</Tabs>

## Additional Identity endpoints

The following endpoints are also registered by `MapIdentityApi` and documented in the Scalar UI. They are part of the standard ASP.NET Core Identity HTTP API:

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/Users/confirmEmail` | Confirms a user's email address using the token sent by email. |
| `POST` | `/api/Users/resendConfirmationEmail` | Sends a new email confirmation link to the specified address. |
| `POST` | `/api/Users/forgotPassword` | Sends a password reset link to the specified email address. |
| `POST` | `/api/Users/resetPassword` | Resets a user's password using the token sent by email. |
| `POST` | `/api/Users/manage/2fa` | Enables, disables, or retrieves two-factor authentication settings. |
| `GET`  | `/api/Users/manage/info` | Returns the current user's email and 2FA status. |
| `POST` | `/api/Users/manage/info` | Updates the current user's email or password. |

<Note>
  The seeded administrator account uses `administrator@localhost` as the email and `Administrator1!` as the password. You can use these credentials to log in on a fresh database without registering first.
</Note>
